# Custom Types in Go

With **custom types**, we can build our own data types and extend them with methods, similar to classes in OOP (though Go does not have classes).

```go
type Deck []string
```

In this line, we define a new type `Deck` based on `[]string`.
This allows us to attach methods to `Deck`.

---

## Receiver Functions (Methods)

In Go, a **receiver** is specified in parentheses before the function name.
It indicates which type the function belongs to.

Functions with receivers are called **methods**.

### Example

```go
type Deck []string

func (deck Deck) print() {
	for i, card := range deck {
		fmt.Println(i, card)
	}
}
```

In this example:

* `deck` is the **receiver**
* `Deck` is the receiver’s **type**
* The `print` method can access the data inside a `Deck` value

This allows us to call the method like this:

```go
myDeck.print()
```




# Type Conversion in Go (Easy & Complete Guide)

This document explains **type conversion in Go** in a **simple, clear, and complete way**, with examples and common mistakes.

---

## 1. What Is Type Conversion?

### Simple Definition

> **Type conversion** means changing a value from **one type** to **another type**.

Go is a **strongly typed language**, so it does **not** convert types automatically.

---

## 2. Why Type Conversion Is Needed

Because Go is strict, this will **NOT work**:

```go
var a int = 10
var b float64 = 2.5

fmt.Println(a + b) // ❌ compile-time error
```

You must **explicitly convert** one type.

---

## 3. Basic Type Conversion Syntax

### General Syntax

```go
T(value)
```

Where:

* `T` = target type
* `value` = value to convert

---

## 4. Integer ↔ Float Conversion

### int → float64

```go
a := 10
b := float64(a)

fmt.Println(b) // 10.0
```

### float64 → int

```go
x := 9.8
y := int(x)

fmt.Println(y) // 9
```

⚠ Important:

* Go **truncates** decimals (no rounding)

---

## 5. Integer Type Conversion

Different integer types must be converted explicitly.

```go
var a int = 10
var b int64 = int64(a)
```

Even though both are integers, Go treats them as **different types**.

---

## 6. String ↔ Number Conversion

Go does **not** allow direct conversion between strings and numbers.

❌ This is invalid:

```go
str := "123"
num := int(str) // ❌ invalid
```

---

### String → Integer (`strconv` package)

```go
import "strconv"

str := "123"
num, err := strconv.Atoi(str)

if err != nil {
    fmt.Println("Conversion failed")
}

fmt.Println(num)
```

---

### Integer → String

```go
num := 123
str := strconv.Itoa(num)

fmt.Println(str)
```

---

### String → Float

```go
str := "3.14"
f, err := strconv.ParseFloat(str, 64)
```

---

## 7. Byte, Rune, and String Conversion

### String → []byte

```go
s := "hello"
b := []byte(s)
```

### []byte → String

```go
s := string(b)
```

---

### Rune Conversion

```go
r := 'A'
fmt.Println(r)        // 65
fmt.Println(string(r)) // A
```

Explanation:

* `rune` represents a Unicode code point

---

## 8. Type Conversion vs Type Assertion

⚠ These are **NOT the same**.

### Type Conversion

```go
float64(10)
```

Used for **concrete types**.

---

### Type Assertion (interfaces only)

```go
var x interface{} = "hello"

s := x.(string)
```

Used when working with **interfaces**.

---

## 9. Common Beginner Mistakes

### ❌ Expecting Automatic Conversion

```go
var a int = 5
var b int64 = 10
fmt.Println(a + b) // ❌ error
```

✔ Correct:

```go
fmt.Println(int64(a) + b)
```

---

### ❌ Losing Precision

```go
x := 9.9
y := int(x)
fmt.Println(y) // 9, not 10
```

---

## 10. Best Practices

* Be **explicit** with conversions
* Convert as **close to usage** as possible
* Avoid unnecessary conversions
* Always handle errors with `strconv`

---
